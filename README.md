# tg-random — бот для записи на футбол и жеребьёвки команд

Telegram-бот для группового чата команды: игроки записываются на игру, а бот автоматически проводит жеребьёвку и публикует составы. Дополнительно есть **mini app** (статический фронтенд) для распределения по командам на одном телефоне — работает отдельно от бота.

## Что умеет бот

- **Создание игры** с временем начала (по умолчанию: «Воскресный футбол», Зелёные vs Красные, 7 на команду).
- **Запись игроков** через `/join` или кнопку «Записаться» в группе.
- **Приватные подтверждения** — ответ при записи видит только записавшийся (личка или всплывающее уведомление).
- **Автожеребьёвка** за N минут до игры (по умолчанию 30).
- **Ручная жеребьёвка** админом через `/draw`.
- **Ручное добавление игроков** без Telegram (`/add Иван, Петя`).
- **Админские команды из лички** — создать игру, добавить игроков, провести жеребьёvку без захода в группу.
- **Ограничение лички** — писать боту могут только участники группы (и админы).
- **Личные уведомления** после жеребьёвки («Ты в команде Зелёные»).

## Архитектура

```
┌─────────────────┐     long polling      ┌──────────────────┐
│  Telegram API   │ ◄──────────────────► │  Bot (Telegraf)  │
└─────────────────┘                       │  src/bot/        │
                                          └────────┬─────────┘
                                                   │
                                          ┌────────▼─────────┐
                                          │  data/store.json │
                                          │  (JSON-хранилище)│
                                          └──────────────────┘

┌─────────────────┐
│  Mini App       │  — статика на Vercel, sessionStorage,
│  webapp/        │    для одного телефона (без синхронизации)
└─────────────────┘
```

| Компонент | Технология | Где работает |
|-----------|------------|--------------|
| Бот | Node.js, Telegraf, node-cron, Luxon | VPS 24/7 |
| Хранилище | JSON-файл | `data/store.json` |
| Mini app | HTML/CSS/JS (ES modules) | Vercel / статический хостинг |

**Бот и mini app не связаны между собой** — это два независимых инструмента для разных сценариев.

## Структура проекта

```
tg-random/
├── src/bot/
│   ├── index.js          # точка входа, команды, middleware
│   ├── config.js         # переменные окружения
│   ├── db.js             # чтение/запись store.json
│   ├── gameService.js    # игры, запись, жеребьёвка
│   ├── messages.js       # тексты сообщений
│   ├── notify.js         # приватные уведомления
│   ├── groupAccess.js    # проверка членства в группе
│   ├── adminContext.js   # админ-команды из лички
│   └── scheduler.js      # cron: автожеребьёвка каждую минуту
├── webapp/               # mini app (опционально)
├── data/store.json       # данные (не в git)
├── scripts/vps-setup.sh  # первичная настройка на VPS
├── ecosystem.config.cjs  # конфиг PM2
├── .env.example
└── package.json
```

### Ключевые модули

| Файл | Ответственность |
|------|-----------------|
| `gameService.js` | CRUD игр, `joinGame`, `runDraw`, балансирующий рандом |
| `scheduler.js` | Планировщик: ищет игры с `drawAt <= now` и проводит жеребьёвку |
| `groupAccess.js` | Проверка `getChatMember`, список разрешённых групп |
| `notify.js` | Ответ только записавшемуся (личка / toast) |

## Настройка

### 1. Создай бота в BotFather

1. `/newbot` — получи `BOT_TOKEN`.
2. `/setprivacy` → **Disable** — бот видит команды в группе.
3. `/setcommands` — список команд (опционально).
4. Добавь бота в групповой чат команды.

### 2. Файл `.env`

Скопируй `.env.example` в `.env`:

```bash
cp .env.example .env
```

| Переменная | Описание |
|------------|----------|
| `BOT_TOKEN` | Токен от BotFather |
| `ADMIN_IDS` | Telegram ID админов через запятую |
| `GROUP_CHAT_IDS` | ID группового чата (для команд из лички) |
| `GROUP_INVITE_LINK` | Ссылка-приглашение в группу (опционально) |
| `TIMEZONE` | Часовой пояс (например `Europe/Moscow`) |
| `DRAW_MINUTES_BEFORE` | За сколько минут до игры проводить жеребьёвку |

**Как узнать свой Telegram ID:** напиши [@userinfobot](https://t.me/userinfobot).

**Как узнать ID группы:** добавь [@userinfobot](https://t.me/userinfobot) в группу или возьми из логов при `/newgame`. ID группы отрицательный, например `-1001234567890`.

> **Важно:** если `ADMIN_IDS` пустой, админом считается каждый пользователь.

## Локальный запуск

Требования: **Node.js 18+**.

```bash
git clone https://github.com/dk-1312/tg-random.git
cd tg-random
npm install
cp .env.example .env
```

## Деплой бота на VPS

Бот должен работать **24/7**. Подходит VPS (JustHost и др.), **не** Vercel.

### Первичная установка

```bash
# на сервере
apt update && apt install -y curl git
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

cd /opt
git clone https://github.com/dk-1312/tg-random.git
cd tg-random
cp .env.example .env
nano .env          # заполни переменные

chmod +x scripts/vps-setup.sh
./scripts/vps-setup.sh
```

Или вручную:

```bash
npm install --omit=dev
mkdir -p data
npm install -g pm2
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup        # выполни предложенную команду systemctl
pm2 save
```

### PM2 — полезные команды

```bash
pm2 status
pm2 logs tg-random-bot
pm2 restart tg-random-bot
pm2 stop tg-random-bot
```

Или через npm-скрипты:

```bash
npm run pm2:start
npm run pm2:restart
npm run pm2:logs
```

## Команды бота

### Для всех (в группе)

| Команда | Описание |
|---------|----------|
| `/join` | Записаться на игру |
| `/leave` | Отменить запись |
| `/players` | Список записавшихся |
| `/results` | Составы после жеребьёвки |
| `/start`, `/help` | Справка |

### Для админа (группа или личка)

| Команда | Пример | Описание |
|---------|--------|----------|
| `/newgame` | `/newgame 19:00` | Создать игру |
| `/newgame` | `/newgame 22.06 19:00` | Игра на дату |
| `/add` | `/add Иван, Петя` | Добавить игроков вручную |
| `/remove` | `/remove Иван` | Удалить вручную добавленного |
| `/draw` | — | Жеребьёвка сейчас |
| `/cancel` | — | Отменить текущую игру |

При создании игры из **лички** объявление публикуется в группе автоматически.

## Сценарий использования

1. Админ создаёт игру: `/newgame 19:00` (в группе или личке).
2. Игроки записываются: `/join` или кнопка «Записаться».
3. Игроки пишут боту `/start` в личке — чтобы получать подтверждения без спама в чат.
4. За 30 минут до игры бот автоматически проводит жеребьёвку.
5. Составы публикуются в группе, каждому игроку — личное сообщение с командой.
6. Админ может добавить игроков без Telegram: `/add Серёга`.

### Жеребьёвка

- Игроки перемешиваются случайно.
- Распределение **балансирующее**: чем больше людей в одной команде, тем ниже шанс попасть в неё снова.
- Если записалось меньше лимита — все распределяются между двумя командами.
- Лишние (сверх 7+7) попадают в **запас**.

## Как дорабатывать

### Добавить новую команду

1. Логику — в `gameService.js` (если нужны данные).
2. Текст ответа — в `messages.js`.
3. Обработчик — в `index.js`:

```js
bot.command("mycommand", async (ctx) => {
  // ...
});
```

4. Обнови `helpText()` в `messages.js`.

### Изменить параметры игры по умолчанию

В `index.js`, команда `/newgame`:

```js
title: "Воскресный футбол",
teamAName: "Зеленые",
teamBName: "Красные",
teamSize: 7,
```

### Изменить алгоритм жеребьёвки

Функция `runDraw()` в `gameService.js`.

### Изменить тексты сообщений

Файл `messages.js` — все шаблоны в одном месте.

### Перейти с JSON на SQLite/PostgreSQL

Замени реализацию в `db.js` и `gameService.js`, интерфейс (`withDb`, `getDb`) можно сохранить.

### Добавить напоминание перед игрой

В `scheduler.js` — второй cron или проверка в существующем: за час до `gameAt` отправить сообщение в группу.

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Бот не отвечает в группе | `/setprivacy` → Disable в BotFather |
| `pm2 status` пустой | `pm2 start ecosystem.config.cjs && pm2 save` |
| `Script not found` | Запускай из `/opt/tg-random`, проверь `ecosystem.config.cjs` |
| Личка не работает | Игрок должен быть в группе; админам нужен `GROUP_CHAT_IDS` |
| Нет личных уведомлений | Игрок должен написать боту `/start` в личке |
| Админ-команды из лички не работают | Заполни `GROUP_CHAT_IDS` в `.env` |
| Жеребьёvка не срабатывает | Проверь `TIMEZONE`, `pm2 logs`, что бот online |

### Просмотр логов

```bash
pm2 logs tg-random-bot --lines 100
```

### Проверка без PM2

```bash
cd /opt/tg-random
node src/bot/index.js
```
